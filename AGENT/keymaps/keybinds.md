# Keybindings

Everything bound on this system, per program. Generated by reading the tracked
configs — it is a snapshot, not a second source of truth: the authority is
always the config file named in each section.

Note on the keyboard: dwl loads **us / colemak_dh** with `caps:swapescape`
(Caps Lock is Escape and vice versa). All letter keys below are named the way
they are **labelled on a Colemak-DH layout**, which is how they are written in
`config.h`.

The bindings are not historical accidents — they are allocated by the cost model
in `design.md` (key position weights + chord penalties), on the standing rule
that **`hjkl` means the same thing everywhere**: `h`/`l` the horizontal axis,
`j`/`k` the vertical one, in dwl, Neovim and mpv alike.

## dwl (compositor) — `config/dwl/config.h`

`MOD` = **Alt**. Compile-time config: changing anything here needs
`./install --dwl` plus a new session.

### Launch / windows

| Keys               | Action                                         |
| ------------------ | ---------------------------------------------- |
| `MOD+Space`        | Terminal (foot)                                |
| `MOD+I`            | App launcher (`rofi -show drun`)               |
| `MOD+W`            | Browser (zen-browser) — _w_eb                  |
| `MOD+O`            | Close the focused window                       |
| `MOD+E`            | Toggle fullscreen                              |
| `MOD+V`            | Toggle floating for the focused window         |
| `MOD+Shift+U`      | Lock the screen (waylock)                      |
| `MOD+Shift+Q`      | Quit dwl — asks for confirmation in rofi first |

Close sits on the right pinky, three columns away from the `hjkl` cluster, so a
navigation slip cannot reach it. Lock is on a key that is **unbound** on the
plain `MOD` layer, so no stray Shift can lock the screen.

### Layout / focus

| Keys          | Action                                              |
| ------------- | --------------------------------------------------- |
| `MOD+J`       | Focus next window in the stack                      |
| `MOD+K`       | Focus previous window in the stack                  |
| `MOD+H`       | Shrink the master area (−2 %)                       |
| `MOD+L`       | Grow the master area (+2 %)                         |
| `MOD+N`       | Zoom — swap the focused window into the master area |
| `MOD+Shift+H` | One fewer window in the master area                 |
| `MOD+Shift+L` | One more window in the master area                  |

`h`/`l` is the horizontal axis: plain it resizes the master area, with Shift it
changes how many windows live in it.

New windows attach at the **bottom** of the stack (`attachbottom.patch`), so
opening one never displaces what you are looking at.

### Tags (9)

| Keys                | Action                             |
| ------------------- | ---------------------------------- |
| `MOD+1` … `9`       | View tag 1–9                       |
| `MOD+Shift+1` … `9` | Move the focused window to tag 1–9 |

### Media / hardware

| Keys     | Action                                                                   |
| -------- | ------------------------------------------------------------------------ |
| `MOD+F1` | Mute toggle (`vol_ctl mute`)                                             |
| `MOD+F2` | Volume −5 % (`vol_ctl down`)                                             |
| `MOD+F3` | Volume +5 % (`vol_ctl up`)                                               |
| `MOD+F5` | Brightness down (`bright_ctl down`)                                      |
| `MOD+F6` | Brightness up (`bright_ctl up`)                                          |
| `Print`  | Select a region → screenshot to the clipboard (`grim`+`slurp`+`wl-copy`) |

### Mouse (with `MOD` held)

| Button | Action            |
| ------ | ----------------- |
| Left   | Move the window   |
| Middle | Toggle floating   |
| Right  | Resize the window |

### Escape hatches

| Keys                  | Action                          |
| --------------------- | ------------------------------- |
| `Ctrl+Alt+F1` … `F12` | Switch to virtual terminal 1–12 |

## Neovim — `config/nvim/`

Leader is `Space`. `Space`, `Backspace` and `F1` are mapped to nothing in normal
mode on purpose, so a stray press does not do something surprising.

### Own mappings — `lua/config/keymaps.lua`

| Keys              | Mode   | Action                                                    |
| ----------------- | ------ | --------------------------------------------------------- |
| `<leader>e`       | n      | Open Oil (file _e_xplorer)                                |
| `J` / `K`         | visual | Move the selection down / up, reindented                  |
| `J`               | n      | Join lines, keeping the cursor where it was               |
| `<C-d>` / `<C-u>` | n      | Half page down / up, recentred                            |
| `<C-f>` / `<C-b>` | n      | Full page down / up, recentred                            |
| `n` / `N`         | n      | Next / previous search hit, recentred and unfolded        |
| `<leader>p`       | x      | Paste over the selection from the yank register (no clobber) |
| `<leader>S`       | n      | Search & replace the word under the cursor                |
| `<leader>sd`      | n      | Put diagnostics into the quickfix list                    |
| `<leader>o`       | n      | Open the link under the cursor in the browser             |
| `j` / `k`         | n      | Move by _display_ line — markdown/text buffers only       |

### Oil (inside the file browser) — `lua/plugins/oil.lua`

| Keys    | Action                                |
| ------- | ------------------------------------- |
| `g?`    | Show Oil's own help                   |
| `<CR>`  | Open the entry                        |
| `<C-s>` | Open in a vertical split              |
| `<C-h>` | Open in a horizontal split            |
| `<C-t>` | Open in a new tab                     |
| `<C-p>` | Preview                               |
| `<C-c>` | Close Oil                             |
| `<C-l>` | Refresh                               |
| `-`     | Up to the parent directory            |
| `_`     | Jump to the current working directory |
| `` ` `` | `cd` into the directory               |
| `~`     | `cd` into it, tab-scoped              |

| `gx`    | Open with the external default app    |

the menu, `<C-n>`/`<C-p>` cycle, `<CR>` accepts, `<Tab>`/`<S-Tab>` jump through
snippet placeholders.

## mpv — `config/mpv/input.conf`

Only these are overridden; everything else is mpv's default (`Space` pause,
`f` fullscreen, `q` quit, `9`/`0` volume, …).

| Keys                     | Action                          |
| ------------------------ | ------------------------------- |
| `h` / `l`                | Seek ∓5 s                       |
| `H` / `L`                | Seek ∓1 s                       |
| `j` / `k`                | Volume ∓5                       |
| `←` / `→`                | Seek ∓5 s                       |
| `Shift+←` / `Shift+→`    | Seek ∓1 s                       |
| `↑` / `↓`                | Volume ±5                       |
| Mouse wheel up / down    | Volume ±5                       |
| Mouse wheel left / right | Seek ±5 s                       |
| `Ctrl+j` / `Ctrl+J`      | Cycle the subtitle track ± (was `j` / `J`) |
| `Ctrl+l` / `Ctrl+L`      | A-B loop / loop file (was `l` / `L`)       |

## Programs running on their defaults

These run on their upstream bindings — listed so it is clear that nothing was
changed, with the handful of keys worth remembering. The one exception is rofi,
where `Ctrl+J`/`Ctrl+K` were added to `config.rasi` for the sake of the
system-wide `hjkl` axes:

| Program   | Config                         | Notable defaults                                                                                                        |
| --------- | ------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| foot      | `config/foot/foot.ini`         | `Ctrl+Shift+C`/`V` copy & paste, `Ctrl+Shift+R` search, `Ctrl+Shift+N` new window, `Ctrl++`/`-` font size               |
| rofi      | `config/rofi/`                 | Type to filter, `↑`/`↓`, `Ctrl+J`/`Ctrl+K` or `Ctrl+N`/`Ctrl+P` to move, `Enter` to run, `Esc` to cancel, `Shift+Enter` runs in a terminal |
| btop      | `config/btop/btop.conf`        | `Esc`/`m` menu, `q` quit, `+`/`-` update interval, `f` filter, `k` kill process                                         |
| waylock   | none (CLI flags in `config.h`) | Type the password, `Enter` to unlock — no other input                                                                   |
| KeePassXC | `config/keepassxc/`            | `Ctrl+C` copy password, `Ctrl+B` copy username, `Ctrl+F` search, `Ctrl+L` lock database                                 |
