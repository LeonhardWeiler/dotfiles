2 · Was schon stark ist

stark install · setup/links.conf

Der eigene Symlink-Manager ist schlank und sicher: eine Mapping-Datei (setup/links.conf, zwei Spalten) plus reines Bash ohne Abhängigkeiten (kein Python/dotbot/Stow). Kommandos link/setup/unlink/status/clean, dazu -n/--dry-run, --user-only und --force (sichert reale Ziele nach .bak); das setup-Kommando bündelt den Erst-Setup (Pakete → Link → Units → locale-gen → Checkliste). Reale Dateien werden nie ungefragt überschrieben, unlink entfernt nur Symlinks auf eigene Quellen, /etc-Ziele laufen dateiweise per sudo, Glob-Quellen (config/usrbin/*) verlinken dateiweise, damit Fremd-Einträge in ~/.local/bin (z. B. claude) erhalten bleiben. Live verifiziert: 40/40 Einträge ok.

stark config/usrbin/lib_hypr.sh · workspace_slf

Die Workspace-Automatisierung ist bemerkenswert robust: Fenster werden über ihre stabile Adresse statt über fragile Titel angesprochen, neue Fenster via Adress-Diff erkannt, es gibt Timeout/Poll-Logik (wait_for_new_window), einen SELF_GUARD gegen das Schließen des eigenen Terminals, idempotentes Verhalten und Fehlermeldungen als Rofi-Popup. Sauber dokumentiert und modular (Lib + Runner).

stark config/usrbin/rofi_keepassxc

Das entschlüsselte Eintrags-Listing wird über einen trap 'rm -f "$CACHE/tmp"' EXIT zuverlässig entfernt — auch bei jedem frühen Abbruch (Escape, falsches Passwort). So bleiben keine Klartext-Metadaten der Passwort-DB liegen. Sauberer Umgang mit sensiblen Zwischendaten.

stark config/git/.gitconfig

SSH-basiertes Commit-Signing (gpg.format = ssh, commit.gpgsign = true, signingkey als id_ed25519.pub) ist eingerichtet — gute Sicherheitshygiene für die Commit-Authentizität.

stark alacritty · mako · rofi · hyprland.lua

Durchgängig konsistentes, dunkles Farbschema (#141414 / #dddddd / Akzent #768e84) über alle UI-Komponenten hinweg — wirkt wie aus einem Guss. Auch der Health-Report selbst nutzt dasselbe Schema.

behalten schlanke, bewusste Configs

Ohne Änderungsbedarf (jede Option trägt): alacritty, mako, rofi, mpv (bewusste Cache-/Screenshot- Profile, fremde Lua-Scripts unangetastet), typst (lokales Paket school-theme), systemd (battery-check-Timer, dotfiles-sync), pacman-Hook (minimaler Trigger). keepassxc: nur Config getrackt, die .kdbx ist gitignored (lokal vorhanden). nvim hat eine eigene CLAUDE.md und wird dort vertieft.

3 · Bewusst entschieden

bewusst /etc/sudoers · %wheel

%wheel ALL=(ALL:ALL) NOPASSWD: ALL — passwortloses sudo für die wheel-Gruppe. Bewusste Bequemlichkeits- Entscheidung (im README unter „Manual system state" festgehalten). Zur Kenntnis: jeder Prozess des Users wird damit ohne weitere Hürde root; die sudo-Datei bleibt auf Wunsch unverändert.

bewusst config/usrbin/workspace_slf:75

workspace_slf startet Claude mit --dangerously-skip-permissions (Bypass-All-Modus) — bewusste Workflow-Entscheidung für schnelles Arbeiten. Zur Kenntnis: läuft ohne Permission-Prompts, nur in vertrauenswürdigen Projektverzeichnissen sinnvoll.

bewusst config/hypr/looknfeel.lua:31 · config/nvim/lua/config/options.lua:3

Animationen in Hyprland deaktiviert (enabled = false in looknfeel.lua) und Maus in nvim ausgeschaltet (vim.opt.mouse = "") — bewusst minimalistischer, tastaturzentrierter Workflow. Die Animations-Kurven bleiben in animations.lua definiert, sodass ein Aktivieren nur das Umschalten des Flags erfordert.

bewusst config/mpv/mpv.conf

Sehr große Demuxer-Caches (mehrere GB Readahead) — laut Kommentar bewusst, „viel RAM vorhanden". Legitim für ein Desktop-Setup mit reichlich Speicher.

bewusst config/usrbin/restic-backup:10

Restic-Repo als SFTP zu einer LAN-IP mit Windows-Pfad (sftp:Leo@192.168.178.23:D:\ResticBackup). Interne IP ist kein Geheimnis; RESTIC_PASSWORD bewusst nicht im Skript — wird interaktiv abgefragt. In Ordnung, solange das Skript nicht non-interaktiv (z. B. per Timer) laufen soll.

bewusst config/mkinitcpio/mkinitcpio.conf:55 · config/vconsole/vconsole.conf

Die Console-Keymap (KEYMAP=mod-dh-iso-uk, Colemak-DH) wird über den initramfs-keymap-Hook geladen (in HOOKS=(… keyboard keymap consolefont …)), nicht (nur) über systemd-vconsole-setup — Letzteres überspringt beim Boot die erste VT („Configuration of first virtual console was skipped"). Der keymap-Hook backt die Keymap zur initramfs-Bauzeit ein. Zur Kenntnis: nach jeder Änderung an vconsole.conf einmal sudo mkinitcpio -P laufen lassen, sonst greift beim ly-/ TTY-Login weiter die alte Keymap.

bewusst config/hypr/env.lua:7 · config/hypr/monitors.lua

Gerätespezifische Hardcodes: WLR_DRM_DEVICES auf einen festen PCI-Pfad (pci-0000:34:00.0-card) sowie feste Monitor-Outputs (eDP-1, HDMI-A-1). Für ein persönliches Single-Machine-Setup in Ordnung — nur als Portabilitäts-Hinweis: bei Hardware-Wechsel müssen diese Werte angepasst werden.
