2 · What's already strong

strong install · setup/links.conf

The custom symlink manager is lean and safe: one mapping file (setup/links.conf, two columns) plus plain Bash with no dependencies (no Python/dotbot/Stow). Commands link/setup/unlink/status/clean, plus -n/--dry-run, --user-only and --force (backs up real targets to .bak); the setup command bundles the first-time setup (packages -> link -> units -> locale-gen -> checklist). Real files are never overwritten unasked, unlink only removes symlinks to our own sources, /etc targets run per file via sudo, glob sources (config/usrbin/*) link per file so foreign entries in ~/.local/bin (e.g. claude) are preserved. Live-verified: 40/40 entries ok.

strong config/usrbin/lib_hypr.sh · workspace_slf

The workspace automation is remarkably robust: windows are addressed by their stable address instead of fragile titles, new windows detected via an address diff, there is timeout/poll logic (wait_for_new_window), a SELF_GUARD against closing your own terminal, idempotent behaviour and error messages as a Rofi popup. Cleanly documented and modular (lib + runner).

strong config/usrbin/rofi_keepassxc

The decrypted entry listing is reliably removed via a trap 'rm -f "$CACHE/tmp"' EXIT - even on any early abort (Escape, wrong password). So no plaintext metadata of the password DB is left behind. Clean handling of sensitive intermediate data.

strong config/git/.gitconfig

SSH-based commit signing (gpg.format = ssh, commit.gpgsign = true, signingkey as id_ed25519.pub) is set up - good security hygiene for commit authenticity.

strong alacritty · mako · rofi · hyprland.lua

A consistently dark colour scheme throughout (#141414 / #dddddd / accent #768e84) across all UI components - feels all of a piece. The health report itself uses the same scheme.

keep lean, deliberate configs

Without any need for change (every option earns its place): alacritty, mako, rofi, mpv (deliberate cache/screenshot profiles, third-party Lua scripts untouched), typst (local package school-theme), systemd (battery-check timer, dotfiles-sync), pacman hook (minimal trigger). keepassxc: only config tracked, the .kdbx is gitignored (present locally). nvim has its own CLAUDE.md and is covered in more depth there.

3 · Machine-specific - won't work as-is for another user

Personal dotfiles: the following values are hardcoded to this exact machine/user (hostname "leo", a specific laptop + LAN). On a fresh Arch install for someone else they are wrong and need adjusting. None of these are bugs - they are listed so a foreign user knows what to change before running ./install.

machine-specific config/git/.gitconfig

Hardcoded git identity and signing key: name = Leonhard Weiler, email = leonhard-weiler@hotmail.com and signingkey = /home/leo/.ssh/id_ed25519.pub with commit.gpgsign = true. A foreign user would commit under my name and, worse, every commit fails to sign because that key path does not exist for them.

Adjust: set your own name/email and point signingkey at your own key (or drop commit.gpgsign).
machine-specific config/hypr/env.lua:7 · config/hypr/monitors.lua

WLR_DRM_DEVICES is pinned to a fixed PCI GPU path (/dev/dri/by-path/pci-0000:34:00.0-card) and the monitors are fixed outputs (eDP-1, HDMI-A-1) with specific resolutions/positions. On different hardware the PCI path does not exist and the outputs are named differently, so Hyprland may pick the wrong GPU or leave monitors unconfigured.

Adjust: set WLR_DRM_DEVICES to your card (or remove it) and rewrite monitors.lua for your outputs (hyprctl monitors).
machine-specific config/vconsole/vconsole.conf

The console keymap is KEYMAP=mod-dh-iso-uk (Colemak-DH). A QWERTY user gets a Colemak TTY and ly login, which is unusable until changed (and baked into the initramfs, see the keymap note under "Deliberate decisions").

Adjust: set your layout in vconsole.conf, then sudo mkinitcpio -P.
machine-specific config/usrbin/restic-backup:10

The restic default repo is sftp:Leo@192.168.178.23:D:\ResticBackup (my LAN NAS, Windows path, user "Leo") and it backs up $HOME/files/projects and $HOME/files/school. For anyone else the target host is unreachable and those source directories don't exist. The repo is overridable via RESTIC_REPOSITORY, but the default and the backed-up paths are mine.

Adjust: set RESTIC_REPOSITORY (env) and change the backup source directories.
machine-specific config/usrbin/workspace_slf

A personal project launcher: it opens the dev layout for $HOME/files/projects/slf (zen browser, nvim, frontend + backend terminals) and aborts if that directory is missing. The whole script is specific to one of my projects and has no meaning on another machine.

Adjust: repurpose or ignore - it only runs when bound to a key and the project path exists.
machine-specific config/bash/.bashrc:7 · config/usrbin/bat_check:3

The battery indicator reads /sys/class/power_supply/BAT0/capacity. Both the prompt (.bashrc) and the low-battery notifier (bat_check) degrade gracefully when it is absent (empty display / exit 0), but on a laptop whose battery is BAT1 the indicator silently shows nothing.

Adjust: point BAT_CAPACITY/CAPACITY at your battery device (desktops need no change).

4 · Deliberate decisions

deliberate /etc/sudoers · %wheel

%wheel ALL=(ALL:ALL) NOPASSWD: ALL - passwordless sudo for the wheel group. Deliberate convenience decision (recorded in the README under "Manual system state"). Note: every process of the user thereby becomes root without any further hurdle; the sudo file stays unchanged on purpose.

deliberate config/usrbin/workspace_slf:75

workspace_slf starts Claude with --dangerously-skip-permissions (bypass-all mode) - deliberate workflow decision for fast work. Note: runs without permission prompts, only sensible in trusted project directories.

deliberate config/hypr/looknfeel.lua:31 · config/nvim/lua/config/options.lua:3

Animations disabled in Hyprland (enabled = false in looknfeel.lua) and the mouse turned off in nvim (vim.opt.mouse = "") - deliberately minimalist, keyboard-centric workflow. The animation curves stay defined in animations.lua, so enabling only requires flipping the flag.

deliberate config/mpv/mpv.conf

Very large demuxer caches (several GB readahead) - deliberate per the comment, "plenty of RAM available". Legitimate for a desktop setup with ample memory.

deliberate config/usrbin/restic-backup

RESTIC_PASSWORD is deliberately not in the script - it is prompted interactively. Fine as long as the script is not meant to run non-interactively (e.g. via a timer). (The repo target and backup paths are machine-specific - see section 3.)

deliberate config/mkinitcpio/mkinitcpio.conf:55 · config/vconsole/vconsole.conf

The console keymap (KEYMAP=mod-dh-iso-uk, Colemak-DH) is loaded via the initramfs keymap hook (in HOOKS=(… keyboard keymap consolefont …)), not (only) via systemd-vconsole-setup - the latter skips the first VT on boot ("Configuration of first virtual console was skipped"). The keymap hook bakes the keymap in at initramfs build time. Note: after every change to vconsole.conf run sudo mkinitcpio -P once, otherwise the old keymap keeps applying at the ly / TTY login.
