2 · What's already strong

strong install · setup/links.conf

The custom symlink manager is lean and safe: one mapping file (setup/links.conf, two columns) plus plain Bash with no dependencies (no Python/dotbot/Stow). Commands link/setup/unlink/status/clean, plus -n/--dry-run, --user-only and --force (backs up real targets to .bak); the setup command bundles the first-time setup (packages -> link -> units -> locale-gen -> checklist). Real files are never overwritten unasked, unlink only removes symlinks to our own sources, /etc targets run per file via sudo, glob sources (config/usrbin/*) link per file so foreign entries in ~/.local/bin (e.g. claude) are preserved. Live-verified: 40/40 entries ok.

strong install · validate_links / ./install validate

The links.conf handling is a strict parse → validate → build → execute pipeline: every command validates first and aborts on any problem (nothing changed), reporting links.conf:<line>: <msg>. It rejects a missing target, stray fields, an absolute or repo-escaping source (realpath vs the repo root), a non-existent source, duplicate sources/targets (expanded), a target outside the allowlist (~ / /etc / /usr/local) and an unmatched glob (optional opts out). A read-only ./install validate and a hidden selftest (links into a throwaway mktemp -d HOME, verifies the symlinks, unlinks and verifies removal - never touching the real home) are both wired into a CI workflow (.github/workflows/validate.yml). Verified: the real config passes, crafted bad configs abort with the right message, and selftest creates and removes 30/30 links.

strong setup/services.txt · groups.txt · fonts.txt

The installer is now almost entirely data-driven: the systemd unit lists (services.txt, <scope> <unit>), the group list (groups.txt) and the font packages (fonts.txt) live in plain setup/*.txt files read by small loaders instead of being hardcoded in the script - matching the existing links.conf/programs.txt pattern. Empty/missing lists degrade gracefully. Combined with --only/--exclude for selective linking, the installer stays generic and easy to retarget.

strong install · do_scrub / --scrub

The non-owner path is a clean addition: the setup command asks "Are you the repository owner?" and, on "no", runs an idempotent scrub over the working copy before linking - blanking the git identity and removing commit signing, replacing the restic backup host with a showcase placeholder (sftp:user@ip:drive/path/to/your/backup) and dropping the WLR_DRM_DEVICES iGPU pin so wlroots auto-detects the GPU. It is a tri-state (--scrub / --no-scrub, also runnable standalone) that honours -n/--dry-run and never scrubs silently in a non-interactive run. This closes the biggest portability gap - a foreign clone no longer deploys the owner's identity, a broken signing key or a wrong GPU path.

strong config/usrbin/rofi_keepassxc

The decrypted entry listing is reliably removed via a trap 'rm -f "$CACHE/tmp"' EXIT - even on any early abort (Escape, wrong password). So no plaintext metadata of the password DB is left behind. Clean handling of sensitive intermediate data.

strong config/git/.gitconfig

SSH-based commit signing (gpg.format = ssh, commit.gpgsign = true, signingkey as id_ed25519.pub) is set up - good security hygiene for commit authenticity.

strong foot · mako · rofi · hyprland.lua

A consistently dark colour scheme throughout (#141414 / #dddddd / accent #768e84) across all UI components - feels all of a piece. The health report itself uses the same scheme.

keep lean, deliberate configs

Without any need for change (every option earns its place): foot, mako, rofi, mpv (deliberate cache/screenshot profiles, third-party Lua scripts untouched), typst (local package school-theme), systemd (battery-check timer, dotfiles-sync), pacman hook (minimal trigger). keepassxc: only config tracked, the .kdbx is gitignored (present locally). nvim has its own CLAUDE.md and is covered in more depth there.

3 · Machine-specific - won't work as-is for another user

Personal dotfiles: the following values are hardcoded to this exact machine/user (hostname "leo", a specific laptop + LAN). On a fresh Arch install for someone else they are wrong and need adjusting. None of these are bugs - they are listed so a foreign user knows what to change before running ./install.

machine-specific config/git/.gitconfig

Hardcoded git identity and signing key: name = Leonhard Weiler, email = leonhard-weiler@hotmail.com and signingkey = /home/leo/.ssh/id_ed25519.pub with commit.gpgsign = true. A foreign user would commit under my name and, worse, every commit fails to sign because that key path does not exist for them. This is now handled automatically: answering "no" to the owner prompt (or ./install --scrub) blanks the identity and removes the signing block before linking.

Adjust: run setup as a non-owner (or --scrub), then set your own name/email and point signingkey at your own key.
machine-specific config/hypr/monitors.lua · config/hypr/env.lua

The monitors are fixed outputs (HDMI-A-1 at 2560x0, eDP-1 at 2560x1600@60, scale 1.6) with specific positions/scales. On different hardware the outputs are named differently, so monitors may be left unconfigured. env.lua also pins WLR_DRM_DEVICES to this machine's integrated GPU (pci-0000:34:00.0) - but that line is auto-removed by the non-owner scrub (owner prompt / --scrub), so on foreign hardware wlroots falls back to auto-detection. Only monitors.lua still needs a manual edit.

Adjust: rewrite monitors.lua for your outputs (hyprctl monitors); the GPU pin is handled by the scrub.
machine-specific config/vconsole/vconsole.conf

The console keymap is KEYMAP=mod-dh-iso-uk (Colemak-DH). A QWERTY user gets a Colemak TTY and ly login, which is unusable until changed (and baked into the initramfs, see the keymap note under "Deliberate decisions").

Adjust: set your layout in vconsole.conf, then sudo mkinitcpio -P.
machine-specific config/usrbin/restic-backup:10

The restic default repo is sftp:Leo@192.168.178.23:D:\ResticBackup (my LAN NAS, Windows path, user "Leo") and it backs up $HOME/files/projects and $HOME/files/school. For anyone else the target host is unreachable and those source directories don't exist. The repo is overridable via RESTIC_REPOSITORY, but the default and the backed-up paths are mine. The host default is scrubbed to a showcase placeholder for non-owners (owner prompt / --scrub); the backup source paths ($HOME/files/projects, $HOME/files/school) are not scrubbed and stay mine.

Adjust: set RESTIC_REPOSITORY (env) and change the backup source directories.

4 · Deliberate decisions

deliberate /etc/sudoers · %wheel

%wheel ALL=(ALL:ALL) NOPASSWD: ALL - passwordless sudo for the wheel group. Deliberate convenience decision (recorded in the README under "Manual system state"). Note: every process of the user thereby becomes root without any further hurdle; the sudo file stays unchanged on purpose.

deliberate config/hypr/looknfeel.lua:31 · config/nvim/lua/config/options.lua:3

Animations disabled in Hyprland (enabled = false in looknfeel.lua) and the mouse turned off in nvim (vim.opt.mouse = "") - deliberately minimalist, keyboard-centric workflow. The animation curves stay defined in animations.lua, so enabling only requires flipping the flag.

deliberate config/mpv/mpv.conf

Very large demuxer caches (several GB readahead) - deliberate per the comment, "plenty of RAM available". Legitimate for a desktop setup with ample memory.

deliberate config/usrbin/restic-backup

RESTIC_PASSWORD is deliberately not in the script - it is prompted interactively. Fine as long as the script is not meant to run non-interactively (e.g. via a timer). (The repo target and backup paths are machine-specific - see section 3.)

deliberate config/mkinitcpio/mkinitcpio.conf:55 · config/vconsole/vconsole.conf

The console keymap (KEYMAP=mod-dh-iso-uk, Colemak-DH) is loaded via the initramfs keymap hook (in HOOKS=(… keyboard keymap consolefont …)), not (only) via systemd-vconsole-setup - the latter skips the first VT on boot ("Configuration of first virtual console was skipped"). The keymap hook bakes the keymap in at initramfs build time. Note: after every change to vconsole.conf run sudo mkinitcpio -P once, otherwise the old keymap keeps applying at the ly / TTY login.
