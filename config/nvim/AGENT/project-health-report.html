2 · What is already strong

Clean module structure

init.lua loads deliberately config.lazy -> options -> keymaps -> autocmds; plugins live individually under lua/plugins/ and are collected automatically via { import = "plugins" }. Clear and easy to maintain.

Robust lazy.nvim bootstrap

The bootstrap in lazy.lua uses vim.uv with a vim.loop fallback, handles clone errors cleanly and disables the annoying change_detection notification.

Modern LSP setup

lsp.lua uses the new vim.lsp.config API (Neovim 0.11+) with per-server configuration; the servers are installed and activated automatically via mason-lspconfig (ensure_installed, mason-org org). Capabilities come from blink.cmp, lazydev.nvim provides Lua with the Neovim runtime types. Keymaps cleanly via an LspAttach autocmd.

Modern editing stack

Completion via blink.cmp (ghost text, signature help, LuaSnip), formatting via conform.nvim (format-on-save with LSP fallback) and linting via nvim-lint. Non-LSP tools are installed automatically by mason-tool-installer. mini.pairs/mini.icons replace older single plugins.

Treesitter on the main branch

nvim-treesitter runs on the future-proof main branch with an explicit parser list and FileType-driven highlighting/indent. Text objects and incremental selection are deliberately not used.

Well-considered editor defaults

options.lua sets sensible foundations: scrolloff=10, an always-visible signcolumn, disabled swap/backup files, consistent 2-space indents, the system clipboard (unnamedplus) and a compiled kanagawa theme.

Good plugin core

oil.nvim as the file manager (with trash support), gitsigns, Telescope, Harpoon, Comment.nvim and typst-preview form a lean, practical selection without overhead.

Consistent formatting & readable diagnostics

A stylua.toml (spaces, width 2) fixes the indentation to match options.lua; stylua --check lua/ is green. Linting runs without false alarms (Lua via lua_ls, no more selene/markdownlint), and diagnostics are shown multi-line under the affected line via virtual_lines - you see the full message, not just a squiggle.

3 · Deliberately decided

Git only as signs, actions in the terminal (UX-3)

gitsigns.nvim deliberately stays without hunk keymaps: the only thing wanted is showing the changed lines in the signcolumn (that runs by default). Git actions (stage/reset/commit/blame) are deliberately done in the terminal, not from within Neovim - so no on_attach is needed.

Mouse disabled

vim.opt.mouse = "" deliberately turns the mouse off entirely - consistent keyboard-first use, not a finding.

hlsearch off, incsearch on

Search matches are not highlighted permanently (hlsearch=false), only incrementally - deliberate calm in the buffer.
